Fix the hero section layout in index.html and styles.css.

GOAL:
The “At a glance” card should never overlap the hero text, and it should remain visible on desktop, tablet, and mobile.

Required behavior:

1. DESKTOP (1280px and above)
- Use a proper 2-column layout
- Left column: eyebrow, headline, body text, CTA buttons, support text
- Right column: “At a glance” card
- Add enough gap between columns so the card never overlaps the text
- Keep both vertically aligned and visually balanced

2. TABLET (768px to 1279px)
- Do NOT hide the “At a glance” card
- Switch to a single-column layout
- Show hero text first
- Show CTA buttons
- Then place the “At a glance” card below the buttons
- Keep spacing generous and clean

3. MOBILE (below 768px)
- Keep single-column layout
- Hero text first
- CTA buttons second
- “At a glance” card third
- Card should be full width or close to full width
- Do NOT hide the card

4. REMOVE ANY RULES THAT HIDE THE CARD
- Remove display:none or visibility:hidden rules for the “At a glance” card on tablet/mobile
- Remove any layout rule that causes overlap by using negative margins, transforms, absolute positioning, or insufficient column spacing

5. CSS IMPLEMENTATION
Use a structure similar to:
- .hero-layout
- .hero-copy
- .hero-card

Suggested desktop CSS:
.hero-layout {
  display: grid;
  grid-template-columns: minmax(0, 1.1fr) minmax(320px, 420px);
  gap: 56px;
  align-items: start;
}

.hero-copy {
  min-width: 0;
}

.hero-card {
  min-width: 320px;
  max-width: 420px;
  width: 100%;
}

Suggested tablet/mobile CSS:
@media (max-width: 1279px) {
  .hero-layout {
    grid-template-columns: 1fr;
    gap: 32px;
  }

  .hero-card {
    display: block;
    max-width: 720px;
    width: 100%;
  }
}

@media (max-width: 767px) {
  .hero-card {
    max-width: 100%;
  }
}

6. KEEP DESIGN
- Preserve the current visual style
- Preserve the hero image background
- Preserve typography and buttons
- Only fix layout and responsive behavior

Final result:
- No overlap on desktop
- Card visible on all breakpoints
- Card moves below content on tablet/mobile instead of disappearing
